# 個人中心帳戶設定

## 功能概述

允許用戶在個人中心編輯自己的基本資訊和密碼。編輯功能內嵌在個人中心頁面中，無需跳轉到其他頁面。

## 用戶故事

**作為一個照護者**  
我想在個人中心修改顯示名稱、性別、郵箱和手機號碼  
以便保持我的個人資訊最新

**作為一個用戶**  
我想修改密碼而不需要輸入舊密碼  
以便在必要時快速更新安全信息

**作為一個管理員**  
我想從個人中心菜單快速進入後台管理系統  
以便提高工作效率

## 功能描述

### 編輯模式

用戶在個人中心點擊「帳戶設定」後，進入編輯模式：

- **顯示名稱** - 必填，用戶的公開顯示名稱
- **性別** - 可選選項（男、女、其他）
- **郵箱** - 可選，修改時檢查唯一性
- **手機號碼** - 可選，自由格式
- **修改密碼** - 可展開的密碼修改區域，只需輸入新密碼（最少 8 字符）

編輯表單嵌入在個人中心頁面中，頂部的「設定」按鈕始終可見，用戶可隨時打開菜單。

### 菜單選項

設定菜單包含三個選項：
- **帳戶設定** - 打開編輯表單
- **進入後台管理** - 僅管理員可見，導航到後台
- **登出** - 清除登入狀態，返回登入頁

### 成功提示

編輯完成後顯示成功通知：「成功 - 個人資料已更新」

編輯失敗時顯示錯誤提示，例如郵箱已被他人使用。

## 業務規則

| 規則 | 說明 |
|------|------|
| 顯示名稱必填 | 用戶必須提供顯示名稱 |
| 郵箱唯一性 | 若修改郵箱，需檢查新郵箱是否已被其他活躍用戶使用 |
| 密碼無需舊密碼 | 修改密碼時只需輸入新密碼，無需驗證舊密碼 |
| 密碼長度限制 | 新密碼最少 8 個字符 |
| 角色權限 | 管理員才能看到「進入後台管理」菜單選項 |
| 認證必需 | 所有操作需要有效的登入狀態（JWT Token） |

## 流程

```
用戶進入個人中心
    ↓
點擊「設定」按鈕
    ↓
選擇「帳戶設定」
    ↓
進入編輯模式（表單在頁面中顯示）
    ↓
修改所需資訊
    ↓
點擊「保存」
    ↓
提交至後端
    ↓
驗證通過 → 顯示成功提示 → 返回查看模式
    ↓ 或
驗證失敗 → 顯示錯誤提示 → 留在編輯模式
```

或用戶可點擊「取消」直接返回查看模式。


## API 合約

### 獲取當前用戶資訊
`GET /api/users/me` - 返回當前登入用戶的基本資訊和角色

### 更新用戶資訊
`PUT /api/users/me` - 接受修改後的用戶資訊，返回更新結果
