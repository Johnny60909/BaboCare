## 新增需求

### 需求：建立帶有特定類型選項的寶寶活動

系統應允許授權的照顧者建立寶寶活動記錄，包括類型（餵奶、進食、換尿布、睡覺、心情）、可選照片、備註及特定類型的選項。

#### 情境：照顧者建立餵奶活動

- **當** 照顧者選擇「餵奶」活動類型並點擊發佈時
- **那麼** 活動被儲存，包含類型=Feeding、時間戳、照顧者 ID 及寶寶 ID

#### 情境：照顧者建立帶有副食品選項的進食活動

- **當** 照顧者選擇「進食」活動類型，選擇「副食品」選項，上傳照片，添加備註並點擊發佈時
- **那麼** 活動被儲存，包含類型=Eating、options={foodType: "supplementary"}、照片 URL 及備註

#### 情境：照顧者建立帶有出遊選項的心情活動

- **當** 照顧者選擇「心情」活動類型，選擇「出遊」選項，上傳照片並點擊發佈時
- **那麼** 活動被儲存，包含類型=Mood、options={moodType: "outing"} 及照片 URL

#### 情境：建立活動需要照片及至少一個必填欄位

- **當** 照顧者嘗試在未上傳照片的情況下發佈活動時
- **那麼** 系統顯示驗證錯誤並阻止提交

#### 情境：建立活動時驗證特定類型選項

- **當** 照顧者選擇「進食」活動類型但選項無效時
- **那麼** 系統顯示驗證錯誤並阻止提交

#### 情境：僅授權的照顧者可以建立活動

- **當** 未經授權的用戶嘗試為寶寶建立活動時
- **那麼** 系統返回 403 Forbidden 錯誤

### 需求：具有結構化選項的活動類型定義

系統應支持五種活動類型，每種都有特定選項：

- **餵奶 (Feeding)**：不區分配方奶或母乳；必填照片，可選備註
- **進食 (Eating)**：副食品或午餐選項；必填照片，可選備註
- **換尿布 (Diaper)**：無特殊選項；必填照片，可選備註
- **睡覺 (Sleep)**：無特殊選項；必填照片，可選備註
- **心情 (Mood)**：出遊、玩耍或不開心選項；必填照片，可選備註

#### 情境：系統接受所有五種活動類型

- **當** 照顧者建立每種類型（餵奶、進食、換尿布、睡覺、心情）的活動時
- **那麼** 所有活動皆以正確的類型標識符固化儲存

#### 情境：心情活動限制為三種心情選項

- **當** 照顧者嘗試建立無效心情類型的活動時
- **那麼** 系統進行驗證並拒絕提交，並顯示錯誤消息

### 需求：建立活動頁面的可訪問性

系統應提供一個可從首頁通過底部導航欄中的「+」按鈕訪問的活動建立頁面。

#### 情境：照顧者導航至建立活動頁面

- **當** 照顧者點擊底部導航欄中的「+」按鈕時
- **那麼** 系統導航至建立活動頁面，其中包含寶寶選擇器和活動類型選項

#### 情境：建立活動頁面顯示寶寶下拉選單

- **當** 建立活動頁面加載時
- **那麼** 頁面顯示下拉選單，列出用戶有權記錄活動的所有寶寶
