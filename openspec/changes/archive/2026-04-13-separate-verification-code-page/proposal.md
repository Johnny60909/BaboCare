## Why

目前的驗證碼輸入流程存在兩個問題：
1. 登入頁面包含驗證碼相關的 UI，造成使用者介面混亂
2. 初次登入時驗證碼 UI 沒有確認按鈕，使用者體驗不佳

統一將驗證碼流程導向一個專門的驗證碼輸入頁面，可以簡化登入流程的邏輯並改善使用者體驗。

## What Changes

- **登入頁面 (LoginPage)**：移除所有驗證碼相關的 UI 元素；添加連結指向驗證碼輸入頁
- **驗證碼輸入頁 (ActivatePage)**：確保該頁面有明確的確認按鈕和完整的驗證流程
- **註冊流程**：初次登入（自行申請）完成帳號資料提交後，直接導向驗證碼輸入頁，而非在登入頁顯示驗證碼 UI

## Capabilities

### New Capabilities

- `verification-code-flow`: 獨立的驗證碼輸入流程，從多個入口點（如登入後、註冊後）統一導向專門的驗證碼頁面

### Modified Capabilities

- `user-auth`: 修改登入流程以整合獨立的驗證碼輸入流程，移除登入頁的驗證碼 UI

## Impact

- **前端**：修改 LoginPage、PendingRegisterPage、ActivatePage 等頁面
- **路由**：確保流程中各頁面間的導航邏輯清晰
- **使用者體驗**：簡化登入流程，提供一致的驗證碼輸入體驗
