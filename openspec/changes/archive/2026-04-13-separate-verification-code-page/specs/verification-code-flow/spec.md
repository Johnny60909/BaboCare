## ADDED Requirements

### Requirement: 用戶可以通過獨立的驗證碼流程啟用帳號

系統前端 SHALL 提供獨立的驗證碼輸入頁面（`/activate` 路由），允許用戶在完成初次申請註冊後輸入邀請碼以啟用帳號。該頁面 SHALL 要求用戶輸入邀請碼並提供明確的確認按鈕（「啟用帳號」）。

#### Scenario: 用戶從郵件連結訪問驗證碼頁面

- **WHEN** 新申請的用戶從通知郵件中點擊連結進入 `/activate`
- **THEN** 系統顯示驗證碼輸入頁面，包含標題、説明文字和邀請碼輸入欄位

#### Scenario: 用戶輸入有效邀請碼

- **GIVEN** 用戶位於 `/activate` 頁面
- **WHEN** 用戶輸入有效的邀請碼並點擊「啟用帳號」按鈕
- **THEN** 系統向後端提交驗證請求，驗證成功後導向 `/login` 頁面並預填用戶名，同時顯示「帳號已成功啟用」訊息

#### Scenario: 用戶輸入無效邀請碼

- **GIVEN** 用戶位於 `/activate` 頁面
- **WHEN** 用戶輸入無效或過期的邀請碼並點擊「啟用帳號」按鈕
- **THEN** 系統顯示錯誤訊息「邀請碼無效或已過期，請向保母確認」，並允許用戶重新輸入

### Requirement: 從初次申請頁面自動導向驗證碼頁面

初次申請用戶在 `/register` 頁面成功提交帳號資料後，系統 SHALL 自動將用戶導向 `/activate` 頁面進行邀請碼驗證，而不是導向登入頁面或顯示內聯的驗證碼 UI。

#### Scenario: 註冊成功後導向驗證碼頁面

- **GIVEN** 新用戶填寫並提交了 `/register` 頁面上的註冊表單
- **WHEN** 後端成功創建待審核帳號並回傳 `pendingUserId`
- **THEN** 前端自動導向 `/activate` 頁面，並將 `pendingUserId` 存儲於瀏覽器狀態中供後續驗證使用

#### Scenario: 驗證碼驗證成功後導向登入頁面

- **GIVEN** 用戶已在 `/activate` 頁面成功輸入並驗證邀請碼
- **WHEN** 後端驗證成功並回傳 `userName`
- **THEN** 前端自動導向 `/login` 頁面，URL 應包含 `activated=1` 參數和 `username` 參數（如 `/login?activated=1&username=john`），登入頁面顯示成功提示並預填用戶名

### Requirement: 登入頁面不含驗證碼相關 UI

登入頁面（`/login` 路由）SHALL 專注於帳號密碼登入，**不應**包含任何驗證碼輸入欄位、驗證碼提交按鈕或驗證碼相關的 UI 元素。如果用戶需要驗證，應通過清晰的連結或指引導向 `/activate` 頁面。

#### Scenario: 登入頁面顯示帳號密碼表單

- **WHEN** 未登入用戶訪問 `/login` 頁面
- **THEN** 頁面顯示帳號、密碼輸入欄位和登入按鈕，不顯示任何驗證碼相關 UI

#### Scenario: 帳號已啟用提示

- **WHEN** 用戶通過 `/login?activated=1` 訪問登入頁面
- **THEN** 頁面顯示綠色成功提示「帳號已成功啟用！請使用帳號密碼登入」，並預填用戶名（如果 URL 包含 `username` 參數）

#### Scenario: 用戶尚未驗證時的提示

- **GIVEN** 用戶嘗試使用未驗證的帳號登入並收到 HTTP 400 錯誤
- **WHEN** 錯誤訊息返回
- **THEN** 系統顯示適當的錯誤提示，建議用戶前往 `/activate` 頁面進行驗證

### Requirement: 驗證碼頁面提供完整的使用者反饋

驗證碼頁面 SHALL 實時提供加載狀態、成功/失敗提示和適當的按鈕狀態管理，確保使用者體驗清晰。

#### Scenario: 提交驗證碼時顯示加載狀態

- **GIVEN** 用戶輸入邀請碼後點擊「啟用帳號」按鈕
- **WHEN** 系統向後端提交驗證請求
- **THEN** 按鈕顯示加載中文字（如「驗證中...」），並禁用進一步點擊

#### Scenario: 驗證失敗時顯示錯誤訊息

- **GIVEN** 後端驗證邀請碼失敗（HTTP 400 或其他錯誤）
- **WHEN** 前端收到錯誤回應
- **THEN** 頁面頂部顯示紅色錯誤訊息，按鈕恢復可點擊狀態，用戶可重新輸入

#### Scenario: 空邀請碼時禁用按鈕

- **GIVEN** 用戶位於驗證碼頁面
- **WHEN** 邀請碼輸入欄為空
- **THEN** 「啟用帳號」按鈕應處於禁用狀態
